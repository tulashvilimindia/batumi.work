## 1. GET /announcement/{id}
**Full URL:** `https://api.p.hr.ge/public-portal/tenant/1/api/v3/announcement/{announcementId}`

**Description:** Retrieves full details of a single job announcement by ID.

**Tested IDs:** 1, 100, 450000, 456000, 456240, 456500, 458546, 458675

**ID Range Discovery (Desktop-Chrome):**
- `/announcement/1` - 200 OK (expired=true, mostly null fields)
- `/announcement/100` - 200 OK (expired=true, mostly null fields)
- `/announcement/450000` - 200 OK (expired=true, contains similarAnnouncements array)
- `/announcement/456000` - 200 OK (expired=true)
- `/announcement/456500` - 200 OK (ACTIVE - Carrefour Georgia job, Mobile Sales Coordinator, salary 1300 GEL)
- Low IDs (1-100): Return 200 but with empty/null fields, expired=true
- High IDs (456000+): Contain actual active job data

### Expired Announcements Behavior

When fetching an expired/deleted announcement (e.g., ID 455000 or 450000):
- Returns `announcementId: 0` and `customerId: 0`
- All content fields are `null`
- `expired: true`
- `publishDate: "0001-01-01T00:00:00"` (default datetime)
- `similarAnnouncements` still populated with active jobs!
- `metaData: null`
